# 테스트 규칙 (모든 코딩 작업에 자동 적용)

## 작성 기준
- 새 기능/버그 수정 시 관련 테스트 함께 작성
- 비즈니스 로직, 유틸, API 핸들러 → 단위 테스트 필수
- 핵심 사용자 플로우(로그인, 결제 등) → E2E 테스트 필수

## 단위 테스트
- 순수 함수 우선 — 입력/출력만 검증
- 경계값, 에러 케이스 포함 (빈 배열, null, 초과값)
- 외부 의존성(API, 결제, 이메일) → mock

## Mock 정책
- DB는 가능하면 실제 연결 (mock DB는 동작 차이 위험)
- 프로덕션 DB 직접 테스트 금지 — 테스트 DB 또는 트랜잭션 롤백
- mock 데이터는 실제 스키마와 일치

## E2E (Playwright)
- 핵심 시나리오만 커버 (전수 X)
- 선택자: `data-testid` 우선

## 구조
- 파일: `__tests__/` 또는 소스 옆 `*.test.ts`
- 테스트명은 행동 기술 ("~하면 ~한다")
- 테스트 간 상태 공유 금지

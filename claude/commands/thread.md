### Step 2 — 감정 + 각도 잡기

**먼저 감정 1개를 골라.** 글의 목적에 따라:

| 목적 | 감정 | 공유 동기 |
|------|------|----------|
| 저장 유도 | 유용함 | "똑똑해 보이고 싶어" |
| 공유 유도 | 공감 | "나만 이런 게 아니었어!" |
| 댓글 유도 | 놀람/호기심 | "이게 진짜야?" |
| 클릭 유도 | 불안 | "나는 괜찮은 건가?" |

**그다음 각도를 잡아.**

**트리거가 있는 경우** (예: "오늘 온보딩 이탈률 보고 충격받음"):
- 그 경험/감정이 글의 출발점
- 읽은 문서에서 그 경험에 반전을 줄 수 있는 개념 1개 연결
- "왜 그런 일이 생기는가?"를 한 줄로 설명할 수 있는 각도를 잡아

**트리거가 없는 경우**:
- 읽은 문서에서 개념 하나를 골라
- "이 개념을 틀린 방향으로 알고 있는 사람이 많다"는 각도로 접근
- 각도 확정 전에 스스로 "이게 반전인가, 아니면 뻔한가?"를 확인해

## 출력 형식

### 본글 (1/3)

**0.5초 안에 시선 잡고, 댓글 누르게 만드는 구조. 최대한 짧고 핵심만.**

```
[후킹 1줄] — 반전, 반박, 또는 논쟁 유발형. 30자 이내. 문장 하나로 끝.

[설명 2~3줄] — 누가, 뭘 했는지 핵심만. 구체적 수치나 사실 1개 포함.

[여운 1줄] — 호기심을 남기는 마무리. 명시적 유도("밑에 써놨어", "댓글 참고") 금지.
```

**핵심 원칙:**
- 본글은 "예고편"이야. 다 설명하지 마. 핵심 팩트만 던지고 끊어.
- 설명이 3줄 넘어가면 댓글로 넘겨. 본글에서 다 풀면 댓글 안 눌러.
- 총 4~5줄. 그 이상 쓰지 마.

**후킹 1줄 유형 3가지 (상황에 맞게 선택):**

1. **반전/반박형** — 통념을 뒤집어서 "엥?" 하게 만듦
   - `"코딩 배워야 하나요?" 이 질문 자체가 틀렸어.`
   - `리텐션 올리고 싶으면 기능 추가부터 하지 마.`

2. **논쟁 유발형** — 도발적 질문이나 단정으로 댓글에서 논쟁이 붙게 만듦. "아직도 ~해?", "~하는 사람 진짜 많은데" 같은 구조.
   - `AI한테 아직도 코드 짜달라고 해?`
   - `사이드 프로젝트 3개월째 안 끝나는 사람 특징 있어.`

3. **팩트 폭격형** — 놀라운 수치나 사실을 던져서 "진짜?" 하게 만듦
   - `직원 2명이 연매출 6천억 찍었어.`
   - `온보딩 3단계에서 유저 40%가 사라져.`

짧고, 끊어. 설명 없이 딱 한 줄.

### 댓글 구성 (2~8개)

댓글 수는 내용의 깊이에 따라 자유롭게 결정해. 단, 아래 규칙을 지켜.

**댓글 수 기준:**
- 2개: 핵심 공개 + 반론 선답으로 충분할 때
- 3~4개: 핵심 공개 + 실전 팁 + 반론 선답
- 5~6개: 사례, 단계별 설명, 추가 인사이트가 필요할 때
- 7~8개: 복잡한 주제를 단계적으로 풀어야 할 때 (배경→구조→사례→교훈 등)

**글 구조 원칙 — 배경지식 없이도 이해되게:**
- 전문 용어나 업계 맥락이 필요한 주제는, 댓글 초반에 "이게 뭔데?" 수준부터 깔아줘
- 본글에서 호기심을 걸고 → 댓글에서 배경부터 차근차근 풀어가는 흐름
- 독자가 해당 분야를 전혀 몰라도 마지막 댓글까지 읽으면 핵심을 이해할 수 있어야 해
- 설명이 길어지면 댓글 수를 늘려서 해결해. 한 댓글에 우겨넣지 마

**각 댓글 공통 규칙:**
- 총 6줄 이내
- 한 줄에 문장 하나
- 마지막은 한 줄 마무리로 끊어

**자주 쓰는 댓글 유형 (골라서 조합):**

핵심 공개 — 본글에서 감춘 정보. 번호 리스트 3개 + 마무리 1줄.
```
1. 항목명
짧은 설명 1줄.

2. 항목명
짧은 설명 1줄.

3. 항목명
짧은 설명 1줄.

[마무리 한 줄]
```

실전 팁 — 바로 쓸 수 있는 팁. 번호 리스트 또는 단계.
```
[상황 설정 1줄]
1순위 — ~
2순위 — ~
3순위 — ~
[마무리 한 줄]
```

반론 선답 — 독자가 가질 반박을 먼저 꺼내고 답해.
```
[예상 반론을 자연스럽게 꺼냄]

[짧은 답 1줄]
[부연 또는 수치 1줄]
[마무리 한 줄]
```

추가 인사이트 — 본글과 다른 각도의 정보.
```
[새로운 시각 1줄]
[설명 2줄 이내]
[마무리 한 줄]
```

## 후킹 전략 × 카피 패턴 조합

**8대 훅 전략** (행동경제학 기반, 2~3개 조합):
단일메시지집중 | 손실회피 | 구체적숫자 | 타겟지목 | 간편성 | FOMO | 인정욕구 | 확실성효과

**9개 카피 패턴** (훅 전략과 자유 조합):

| 패턴 | 전략 조합 예시 | 완성 문장 예시 |
|------|-------------|--------------|
| 문제점 후벼파기 | 타겟지목 + 손실회피 | `기획자인데 이 개념 모르면 진짜 손해야.` |
| 비교/경쟁 자극 | FOMO + 구체적숫자 | `같은 3년 차인데 왜 쟤만 연봉이 올랐을까.` |
| 경고/파국 암시 | 손실회피 + 구체적숫자 | `온보딩 3단계에서 유저 40%가 떠나는 이유 있어.` |
| 이익 약속 | 확실성효과 + 간편성 | `이 루틴 하나로 기획 시간 절반 줄어.` |
| 호기심 유발 | 인정욕구 + 단일메시지 | `유저가 앱 지우는 진짜 이유, 아무도 안 말해줘.` |
| 해결책 제시 | 간편성 + 확실성효과 | `퇴사 전에 이것만 준비하면 6개월은 버텨.` |
| 질문/테스트 유도 | 타겟지목 + FOMO | `이 3가지 중 2개 이상이면, 번아웃이야.` |
| 행동 촉구 | 단일메시지 + 간편성 | `스크롤 멈추고 30초만 읽어봐.` |
| 반전/반박 | 손실회피 + 호기심 | `"A 해야 하나요?" 이 질문 자체가 틀렸어.` |

## 품질 체크 (출력 직전 확인)

- [ ] 감정 1개가 명확하고, 글 전체가 그 감정을 관통하는가
- [ ] 첫 줄이 30자 이내이고, 반전/반박 구조인가
- [ ] 본글이 4~5줄 이내인가 (빈 줄 제외)
- [ ] 댓글 1이 번호 리스트 3개 + 마무리 1줄인가
- [ ] 댓글 2가 6줄 이내인가
- [ ] 한 줄에 문장이 2개 이상 들어가지 않았는가
- [ ] 개념 이름을 나열하지 않고 이야기로 녹였는가
- [ ] '~했음' 어조, 이모티콘 없는가

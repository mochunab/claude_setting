# 범용 성장 공식 전략서

> **본능으로 끌어들이고, 감정으로 멈추게 하고, 논란으로 퍼뜨리고, 상품으로 전환하고, 중독으로 잠근다.**
>
> 이 문서는 7대 죄악 프레임워크, 콘텐츠×상품 전략, 바이럴 제작 전략, 바이럴 성장 플레이북 4개 전략서의 교차점에서 추출한 **분야 무관 범용 성장 공식**이다.
> 운세든 F&B든 SaaS든 교육이든, 사람이 고객인 한 이 구조는 작동한다.

---

## 핵심 공식: 5단계 성장 엔진

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ① 본능 설계 (Instinct)    — 왜 관심을 갖는가              │
│       ↓                                                     │
│  ② 훅 설계 (Hook)          — 왜 3초 안에 멈추는가          │
│       ↓                                                     │
│  ③ 확산 설계 (Virality)    — 왜 남에게 퍼뜨리는가          │
│       ↓                                                     │
│  ④ 전환 설계 (Conversion)  — 왜 돈을 내는가                │
│       ↓                                                     │
│  ⑤ 잠금 설계 (Lock-in)    — 왜 떠나지 못하는가             │
│                                                             │
│  ⑤ → ① 로 복귀 (성장 플라이휠)                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**각 단계가 하나라도 빠지면**:
- ① 없으면 → 관심 자체가 발생하지 않음
- ② 없으면 → 유입돼도 이탈
- ③ 없으면 → 광고비 없이는 성장 불가
- ④ 없으면 → 트래픽만 있고 매출 없음
- ⑤ 없으면 → 매출은 있지만 매달 새 고객을 구해야 함

## Stage 1. 본능 설계 — "왜 관심을 갖는가"

### 원리

사람은 논리가 아니라 **본능**으로 움직인다. 글로벌 100대 기업의 이면에는 도덕이 아니라 원초적 욕구가 구매 동기로 작용한다.

### 7대 본능 요약

| 본능 | 핵심 욕구 | 비즈니스 적용 |
|------|----------|-------------|
| **오만 (Pride)** | 나는 특별하다 | 한정판, 프리미엄, 배타적 멤버십 |
| **시기 (Envy)** | 나도 저렇게 | FOMO 마케팅, 인플루언서, 리셀 |
| **분노 (Wrath)** | 이건 부당하다 | 중간상인 제거, 독과점 파괴 |
| **나태 (Sloth)** | 내 시간을 아껴줘 | SaaS, 배달, AI 자동화 |
| **탐욕 (Greed)** | 더 벌고 싶다 | 핀테크, 재테크, 성공학 |
| **탐식 (Gluttony)** | 더 자극적인 것 | F&B, 콘텐츠 플랫폼, 배달 |
| **음욕 (Lust)** | 매력적으로 보이고 싶다 | 뷰티, 피트니스, 데이팅 |

### 킬러 조합 — 본능은 반드시 2개 이상 교차시켜라

단일 본능은 약하다. **2개 이상을 교묘하게 결합할 때 폭발적 시너지**가 발생한다.

| 조합 | 메커니즘 | 예시 |
|------|---------|------|
| 나태 + 탐식 | 노력 없이 쾌락 | 배달앱, OTT |
| 시기 + 오만 | 비교 → 과시 소비 | 명품, SNS |
| 분노 + 나태 | 불편함을 쉽게 해결 | 우버, 토스 |
| 탐욕 + 나태 | 자동으로 돈 벌기 | 로보어드바이저, AI SaaS |
| 음욕 + 시기 | 매력 경쟁 | 데이팅앱, 뷰티 |
| 오만 + 탐욕 | 엘리트 네트워크 | 프리미엄 금융, 블랙카드 |

> **3개 이상 결합 = 메가 플랫폼**
> 인스타그램(음욕+시기+오만+나태), 아마존(나태+탐식+탐욕+분노)

### 실전 적용법

```
Step 1. 우리 제품이 건드리는 본능을 나열한다
Step 2. 2개 미만이면 → 추가 본능을 결합할 수 있는 기능/메시지를 설계한다
Step 3. "왜 이걸 사야 하는가?"의 답이 본능 언어로 나와야 한다
```

**자가 진단**:
- [ ] 우리 제품은 최소 2개 이상의 본능을 자극하는가?
- [ ] 모방 모델(롤모델)이 설계되어 있는가? (모방 욕망 이론)
- [ ] "문제를 인식시키는 언어"가 존재하는가? (PAS의 Agitation)

## Stage 2. 훅 설계 — "왜 3초 안에 멈추는가"

### 원리

사용자는 **3초 안에 볼지 말지를 결정**한다. 본능을 건드리는 것만으로는 부족하고, 그 본능을 **즉각적으로 발화시키는 트리거**가 필요하다.

### 감정 먼저 정하라

콘텐츠의 첫 단계는 주제 선정이 아니라 **"어떤 감정을 건드릴 것인가"**를 정하는 것이다.

| 감정 | 사용자 반응 | 공유 동기 | 포맷 |
|------|-----------|----------|------|
| **공감** | "나만 이런 게 아니었어!" | 관계 유지, 정체성 표현 | 밈, 상황극 |
| **유용함** | "이건 꼭 써먹어야지" | 똑똑한 사람으로 보이고 싶음 | 리스트, 가이드 |
| **놀람** | "와, 이게 진짜?" | 화제성 | 반전 영상, 팩트 |
| **유머** | "친구한테 보내야 함" | 웃음 공유 | 짧은 밈, 패러디 |
| **불안/호기심** | "나는 괜찮은 건가?" | 자기 확인 욕구 | 테스트, 진단 |
| **위로** | "마음이 좀 놓인다" | 감정적 연대 | 고백형, 깨달음형 |

### 감정 흐름 설계

하나의 콘텐츠 안에서 **감정이 이동**해야 몰입이 생긴다:

```
호기심(제목) → 불안/공감(문제 제시) → 놀람/통찰(반전) → 위로/안도(해결)
```

### 행동경제학 기반 8대 훅 전략

| # | 전략 | 핵심 | 예시 |
|---|------|------|------|
| 1 | **단일 메시지 집중** | 한 가지만 전달 | "지금 가입하면 무조건 1만원" |
| 2 | **손실 회피** | 잃는 고통 > 얻는 기쁨 (2배) | "이대로 가면 망함" |
| 3 | **구체적 숫자** | 모호함 = 의심, 숫자 = 신뢰 | "매출 198% 상승시킨 비법" |
| 4 | **타겟 지목** | 칵테일 파티 효과 | "30대 직장인인데 아직도..." |
| 5 | **간편성** | 진입 장벽 최소화 | "딱 3초면", "클릭 한 번" |
| 6 | **FOMO** | 소외 불안 | "요즘 다 아는데 나만 모르는" |
| 7 | **인정욕구** | 우월감, 특별함 | "상위 1%만 아는" |
| 8 | **확실성 효과** | 작아도 확실한 보상 | "무조건 1만원 지급" |

### 본문 3단 구조 (이탈 방지)

| 단계 | 역할 | 예시 |
|------|------|------|
| **1. 문제 제시** | 훅을 구체적 상황으로 확장 | "매일 운동하는데 체중이 그대로?" |
| **2. 반전/인사이트** | 예상을 뒤집는 통찰 | "체중 정체의 80%는 수면 부족" |
| **3. 해결 + CTA** | 솔루션 → 행동 유도 | "오늘부터 이것만 바꿔보세요" |

### CTA 원칙

- 감정 최고조 **직후** 배치 — 1초라도 늦으면 이탈
- **20자 이내** — 동사 하나 + 보상 하나
- 동사를 경량화 — "가입" → "시작하기", "신청" → "확인하기"
- 확실한 보상 명시 — "뭔가 좋은 것" (X) → "무료 진단 결과" (O)

### 실전 적용법

```
Step 1. 감정 1개를 정한다 (감정 표 참고)
Step 2. 8대 전략에서 2~3개를 골라 훅 멘트를 만든다
Step 3. 본문을 3단 구조(문제→반전→해결)로 작성한다
Step 4. 감정 최고조 직후에 CTA를 꽂는다
```

**자가 진단**:
- [ ] 건드릴 감정 1개가 명확한가?
- [ ] 3초 안에 멈추게 하는 훅이 있는가?
- [ ] 본문이 문제→반전→해결 3단 구조인가?
- [ ] CTA가 감정 최고조 직후, 20자 이내인가?

## Stage 3. 확산 설계 — "왜 남에게 퍼뜨리는가"

### 원리

바이럴은 콘텐츠의 품질이 아니라 **사회적 에너지의 설계**다. 사람들이 "소비"만 하고 "공유"하지 않으면 광고비 없이는 성장할 수 없다.

### 바이럴 삼각형

```
        [차별화 소재]
           /\
          /  \
         /    \
        /      \
       /________\
[논란·긴장]  [밈 생산성]
```

세 꼭짓점이 **동시에 작동**할 때 폭발적 바이럴이 발생한다.

| 요소 | 정의 | 없으면? |
|------|------|---------|
| **차별화 소재** | 기존에 없던 조합 | 관심 자체가 발생하지 않음 |
| **논란·긴장** | 찬반이 갈리는 요소 | 화제성이 단발로 끝남 |
| **밈 생산성** | 유저가 갖고 놀 수 있는 구조 | 소비만 되고 확산되지 않음 |

### 3-1. 소재 차별화 공식

```
[익숙한 장르 A] + [예상 밖의 요소 B] = "이런 건 처음이다"
```

| A (익숙함) | B (예상 밖) | 결과 |
|-----------|-----------|------|
| 요리 경연 | 계급 전쟁 | 흑백요리사 |
| 데스게임 | 전래놀이 | 오징어 게임 |
| 재테크 교육 | AI 자동화 | "자동 수익 봇" |
| 다이어트 | DNA 유전자 | "유전자 맞춤 식단" |

**원칙**: A만 = "또 그거", B만 = 대중성 부족. **교집합이 없어 보이는 두 요소의 결합**이 핵심.

### 3-2. 논란 설계 스펙트럼

```
[안전] ←────────────────────────────→ [위험]

의견 갈림      불편한 진실     윤리적 경계선     사회적 금기
(편 가르기)    (통념 뒤집기)   (어디까지 OK?)    (선 넘음)
```

- **스윗 스팟**: "불편한 진실 ~ 윤리적 경계선" 사이
- 왼쪽 끝 = 안전하지만 화제성 약함
- 오른쪽 끝 = 보이콧·규제 리스크

**논란이 바이럴로 전환되는 3가지 조건**:

| 조건 | 없으면? |
|------|---------|
| 콘텐츠 자체 완성도 | 논란으로 유입돼도 악평만 남음 |
| 논란과 핵심 주제의 연결 | 단순 사생활 논란은 시청 동기 약함 |
| 적절한 강도 | 윤리선 넘으면 보이콧 |

### 3-3. 밈 생산성 체크리스트

| 항목 | 높음 | 낮음 |
|------|------|------|
| 모방 용이성 | 특별한 도구 없이 따라 가능 | 특수효과 필요 |
| 캐릭터 식별성 | 별명·캐치프레이즈 있음 | 구분 어려움 |
| 편 가르기 구조 | 정답 없는 토론 가능 | 결론 명확 |
| 명대사/명장면 | 짧은 클립으로 통함 | 맥락 없이 이해 불가 |
| 참여형 포맷 | 예측·투표·테스트 가능 | 일방적 감상만 |

### 3-4. 이중 재활용 — 내부 OSMU + 외부 밈

| 구분 | 설명 | 설계 포인트 |
|------|------|-----------|
| **내부 OSMU** (내가 하는 재활용) | 1소재 → 영상, 카드뉴스, 스레드, 뉴스레터 | 제작 시 "5개 포맷 변환 가능한가?" 체크 |
| **외부 밈** (유저가 하는 재활용) | 챌린지, 패러디, 편 가르기 논쟁 | 참여 장벽 낮추기 + 상징물 + 캐치프레이즈 |

```
내부 OSMU 파이프라인:
영상 → 숏폼(릴스/틱톡) → 주요 장면 캡처 → 카드뉴스
  → 텍스트 요약 → 스레드/커뮤니티 → 인사이트 한 줄 → 뉴스레터
```

### 논란의 자기 강화 루프 (Flywheel)

```
논란 발생
  → 기사·커뮤니티 폭발 (무료 노출)
  → "직접 확인해보자" 시청 유입
  → 알고리즘이 화제작으로 추천
  → 더 많은 노출 → 더 많은 논란
  → 루프 반복
```

### 플랫폼별 역학

| 플랫폼 | 핵심 메커니즘 | 최적 포맷 |
|--------|-------------|----------|
| **틱톡** | 15~60초 + 챌린지 = 참여 폭발 | 숏폼 밈 |
| **유튜브** | 리액션·분석 영상 = 2차 화제성 | 해설/리액션 |
| **인스타/스레드** | 명장면 캡처 + 짧은 의견 = 밈 유통 | 카드뉴스/텍스트 |
| **커뮤니티** | 찬반 댓글 → 트렌딩 → 기사화 | 논쟁 게시글 |
| **뉴스레터/카카오** | 알고리즘 독립 자산, 열람률 80%+ | 깊은 인사이트 |

### 실전 적용법

```
Step 1. 익숙한 A + 예상 밖 B를 조합해 "이런 건 처음이다" 반응을 설계한다
Step 2. 찬반이 갈리는 논란 포인트를 스윗 스팟 범위에서 설정한다
Step 3. 유저가 갖고 놀 수 있는 요소(챌린지/상징물/캐치프레이즈)를 3개 이상 넣는다
Step 4. 내부 OSMU 5개 포맷 + 외부 밈 구조를 동시에 설계한다
```

**자가 진단**:
- [ ] "이런 건 처음이다" 반응이 나오는 차별화 조합인가?
- [ ] 찬반이 갈릴 요소가 있되, 윤리선을 넘지 않는가?
- [ ] 유저가 따라하거나 갖고 놀 수 있는 요소가 3개 이상인가?
- [ ] 1개 소재를 5개 채널로 재활용할 수 있는가?

## Stage 4. 전환 설계 — "왜 돈을 내는가"

### 원리

바이럴이 아무리 터져도 **상품 사다리가 없으면 수익은 0**이다. 유입된 트래픽을 돈으로 바꾸는 구조가 필요하다.

### 콘텐츠 퍼널 — 사람을 모으는 구조

```
[유입] SNS 숏폼 콘텐츠 (스레드/릴스/블로그)
   ↓  도달률 높지만 휘발성
[육성] 뉴스레터 or 카카오 채널 (알고리즘 독립 자산)
   ↓  신뢰 축적 — 구매 전 평균 11.4개 콘텐츠 소비 (HubSpot)
[전환] 상품 랜딩페이지
```

**핵심**: 유입 채널(SNS)은 휘발성이므로, 반드시 **알고리즘에 종속되지 않는 자산**(뉴스레터, 카카오 채널)으로 이동시켜야 한다.

### 채널별 선택 기준

| 채널 | 강점 | 약점 | 적합한 사람 |
|------|------|------|------------|
| **스레드** | 텍스트 도달률 높음, 초기 성장 유리 | 한국 DAU 아직 낮음 | 글 잘 쓰는 사람 |
| **인스타 릴스** | 15초로 대량 노출 | 영상 제작 부담 | 비주얼 있는 분야 |
| **블로그/SEO** | 검색 유입 = 장기 자산 | 초기 3~6개월 느림 | 정보성 콘텐츠 |
| **유튜브** | 신뢰도 최강 | 얼굴 노출 + 편집 부담 | 말 잘하는 사람 |
| **카카오 채널** | 도달률 88%, 열람률 80%+ | 친구 확보 비용 | 한국 타겟 서비스 |

> 유튜브가 부담스러우면 뉴스레터부터. "유튜브 대본 쓴다고 생각하면 쉽다"

### 상품 3단계 사다리

| 단계 | 가격 | 목적 | 예시 |
|------|------|------|------|
| **무료** | 0원 | 가치 증명 + 연락처 수집 | PDF 가이드, 무료 체험, 체크리스트 |
| **저가** | 1만~5만원 | 첫 결제 허들 낮추기 | 템플릿, 미니 강의, 1회 서비스 |
| **고가** | 10만~50만원+ | 핵심 수익원 | 코칭, 풀 강의, 컨설팅, SaaS |

### PAS 공식 — 문제를 "인식시키는" 구조

본능(Stage 1)을 건드려서 유입시킨 후, **"당신에게 이 문제가 있다"고 언어화**해주는 순간 전환이 발생한다.

| 단계 | 역할 | 활용 본능 |
|------|------|----------|
| **Problem** | 고객이 몰랐던 문제를 발견하게 함 | 시기, 분노 |
| **Agitation** | 방치 시 얼마나 나빠지는지 보여줌 | 오만(체면 손상), 시기(격차 벌어짐) |
| **Solution** | 우리 제품만이 유일한 해답 | 나태(쉬운 해결), 탐욕(이득) |

### 가격 전략 4가지

| 전략 | 방법 | 효과 |
|------|------|------|
| **앵커링** | 고가 먼저 보여주고 저가 제시 | 저가 구매율 상승 |
| **마이크로트랜잭션** | 1,000~5,000원 소액 결제 | 충동구매 유도 (전체 결제의 72%) |
| **번들링** | 개별 3개 = 15만 → 패키지 9만 | 객단가 상승 |
| **Opt-out Trial** | 7일 무료 → 자동 유료 전환 | 전환율 49.9% (프리미엄 3~5% 대비 10배) |

### 모방 욕망 이론 — 전환의 숨은 가속기

> 인간은 자신이 원하는 것을 스스로 결정하지 못한다. **타인이 원하는 것을 원한다.** (르네 지라르)

```
모방 모델(롤모델) → 욕망 전염 → 경쟁 → 구매
```

- 인플루언서가 쓰면 → 나도 원함 (욕망 전염)
- 한정판이면 → 경쟁 격화 → 가격 상관없이 구매
- **설계 원칙**: 의도적으로 "모방 모델"을 배치하면 욕망이 자동 생성됨

### 전환율 벤치마크

| 모델 | 설명 | Good | Great | 출처 |
|------|------|------|-------|------|
| **콘텐츠 퍼널** | 뉴스레터 육성 → 상품 구매 | 2~5% | 8~12% | Lenny's Newsletter |
| **프리미엄** | 무료 사용 → 자발적 유료 전환 | 3~5% | 6~8% | First Page Sage 2026 |
| **Opt-out Trial** | 7일 무료 → 자동 유료 전환 | 30%+ | 49.9% | Guru Startups |

> 같은 트래픽이라도 모델에 따라 전환율이 **10배** 차이난다.
> 가능하면 Opt-out Trial(무료체험 → 자동전환)을 우선 설계하라.

### 실전 적용법

```
Step 1. 콘텐츠 퍼널(유입→육성→전환)을 설계한다
Step 2. 무료→저가→고가 상품 사다리를 구축한다
Step 3. 각 단계에 PAS(문제→증폭→해결) 메시지를 배치한다
Step 4. 모방 모델(사용 후기, 인플루언서, 케이스 스터디)을 설계한다
```

**자가 진단**:
- [ ] 유입→육성→전환 퍼널이 연결되어 있는가?
- [ ] 무료→저가→고가 상품 사다리가 존재하는가?
- [ ] 각 단계마다 "이 문제를 방치하면 이렇게 된다"는 메시지가 있는가?
- [ ] 모방 모델(다른 사람도 쓰고 있다)이 설계되어 있는가?

## Stage 5. 잠금 설계 — "왜 떠나지 못하는가"

### 원리

유입과 전환에 성공해도, **이탈을 막지 못하면 매달 새 고객을 구해야 한다**. 쓸수록 떠나기 어려운 구조가 필요하다.

### Hook Model — 중독 루프

```
Trigger(자극) → Action(행동) → Variable Reward(가변 보상) → Investment(투자)
     ↑                                                          |
     └──────────────────────────────────────────────────────────┘
```

| 단계 | 설명 | 설계 포인트 |
|------|------|-----------|
| **External Trigger** | 본능을 건드리는 외부 자극 | 알림, 광고, 이메일 |
| **Internal Trigger** | 감정 자체가 트리거 (습관) | 불안하면 앱을 연다 |
| **Variable Reward** | 예측 불가능한 보상 = 도파민 | 슬롯머신 원리 |
| **Investment** | 사용자가 투자할수록 떠나기 어려움 | 데이터, 팔로워, 히스토리 |

> **핵심**: Trigger 단계에서 **본능 2개 이상**을 동시에 건드리면 Action 전환율이 폭발한다.

### Lock-in + Escalation — 이탈 방지 구조

```
① 본능 자극 (유입)
     ↓
② 데이터/관계/자산 축적 (Investment)
     ↓
③ 전환비용 증가 (Lock-in)
     ↓
④ 본능 강도 증가 (Escalation) — 더 강한 자극 필요
     ↓
⑤ ①로 복귀 (중독 루프 완성)
```

**Lock-in 유형**:

| 유형 | 메커니즘 | 예시 |
|------|---------|------|
| **데이터 잠금** | 사용할수록 추천이 정확해짐 | 넷플릭스, 스포티파이 |
| **관계 잠금** | 팔로워/친구 = 사회적 자산 | 인스타, 카카오톡 |
| **생태계 잠금** | 연동된 서비스가 많을수록 탈출 비용 증가 | 애플, 구글 |
| **학습 잠금** | 사용법을 배우는 데 투자한 시간 | 어도비, 노션 |
| **콘텐츠 잠금** | 쌓인 콘텐츠/기록이 이전 불가 | 블로그, 일기 앱 |

### 리텐션 콘텐츠 주기

```
일간 콘텐츠 (습관 형성)
  ↓
주간 콘텐츠 (깊은 가치)
  ↓
월간 콘텐츠 (변화 추적)
  ↓
시즌 콘텐츠 (바이럴 트리거)
  ↓
→ 재구매 / 업셀 / 추천
```

### 리텐션 채널 효과

| 채널 | 열람률 | 클릭률 | 비용 |
|------|--------|--------|------|
| 카카오 알림톡 | **80%+** | 15~30% | 7~10원/건 |
| 푸시 알림 | 40~60% | 5~15% | 무료 |
| 이메일 뉴스레터 | 20~30% | 2~5% | 무료~저가 |

### AI 개인화 효과

| 지표 | 개선폭 | 출처 |
|------|--------|------|
| 리텐션 | +10~18% | DZ Insights 2025 |
| 세션 시간 | +60% | Apptunix 2026 |
| 재구매율 | 2~3배 | McKinsey |
| 매출 | +10~15% | McKinsey |

### 실전 적용법

```
Step 1. Hook Model 4단계를 설계한다 (Trigger→Action→Reward→Investment)
Step 2. 최소 1가지 이상의 Lock-in 유형을 구현한다
Step 3. 일간/주간/월간/시즌 리텐션 콘텐츠 주기를 설정한다
Step 4. AI 개인화로 "쓸수록 정확해지는" 경험을 만든다
```

**자가 진단**:
- [ ] 사용할수록 떠나기 어려운 Lock-in 구조가 있는가?
- [ ] Variable Reward(예측 불가능한 보상)가 설계되어 있는가?
- [ ] 일간→주간→월간 리텐션 루프가 작동하는가?
- [ ] 더 강한 자극으로의 에스컬레이션(업셀/크로스셀)이 있는가?

## 프랙탈 원리 — PAS는 모든 스케일에서 반복된다

같은 "불편함을 건드리고 → 증폭하고 → 해결"이라는 원리가 콘텐츠 한 개부터 사업 전체까지 **프랙탈처럼 반복**된다.

| 스케일 | 적용 |
|--------|------|
| **콘텐츠 1개** | 훅(문제) → 본문(증폭) → CTA(해결) |
| **캠페인 전체** | 논란(문제 인식) → 무료 노출(증폭) → 콘텐츠 품질로 잔존(해결) |
| **비즈니스 모델** | 시장의 문제를 언어화 → 본능 증폭 → 유일한 해결책 = 우리 제품 |

이 원리를 이해하면, **개별 콘텐츠를 만들 때도 사업 전략을 세울 때도 같은 사고 구조**로 일관되게 설계할 수 있다.

| 본능 조합 | 일상 소재 | 사회 이슈 | 비주류 문화 | 신체/능력 |
|----------|----------|----------|-----------|----------|
| **나태+탐식** | 배달앱 | ? | ? | ? |
| **시기+오만** | 명품 리셀 | 학벌/연봉 비교 | ? | ? |
| **분노+나태** | ? | 세금 환급 서비스 | ? | ? |
| **탐욕+나태** | ? | ? | AI 자동 수익 | ? |
| **음욕+시기** | 뷰티 앱 | ? | ? | DNA 매력 분석 |

> **`?`로 표시된 빈 칸이 아직 시도되지 않은 블루오션이다.**

### 사용법

```

```

| 7 | 유저가 갖고 놀 수 있는 요소가 3개 이상인가? | 3. 확산 |
| 8 | 무료→저가→고가 상품 사다리가 연결되어 있는가? | 4. 전환 |
| 9 | 쓸수록 떠나기 어려운 Lock-in이 있는가? | 5. 잠금 |
| 10 | 1개 소재를 5개 채널로 재활용할 수 있는가? | 3+4 |

> **10개 중 7개 이상 체크되면 성장 엔진이 작동한다.**
> **5개 미만이면 구조적으로 성장이 막힌다.**

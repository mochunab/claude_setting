# 범용 AARRR 퍼널 전략서

> 마케팅과 제품을 분리하지 않는다. 퍼널의 모든 단계에서 "제품이 곧 마케팅"이 되도록 설계한다.
> 어떤 서비스(SaaS, 커머스, 콘텐츠, 앱)에든 적용 가능한 AARRR 프레임워크.
> 최종 업데이트: 2026-04-03

---

## 퍼널 전체 구조

```
Acquisition (유입)
  "서비스를 처음 알게 되는 순간"
    ↓
Activation (활성화)
  "들어와서 '이거 괜찮은데?' 하는 순간"
    ↓
Retention (리텐션)
  "다시 돌아오는 이유가 생기는 순간"
    ↓
Revenue (매출)
  "돈을 쓸 만큼 가치를 느끼는 순간"
    ↓
Referral (추천)
  "친구한테 '너도 해봐' 하는 순간"
    ↓
  → Acquisition으로 복귀 (플라이휠)
```

---

## 1. Acquisition — 유입

> 목표: 서비스를 모르던 사람이 처음 링크를 클릭하게 만든다.

### 채널 선택 프레임워크

| 채널 유형 | 특성 | 적합한 서비스 |
|----------|------|-------------|
| **숏폼 (릴스/틱톡)** | 바이럴/공유, 대량 노출 | 비주얼 결과물이 있는 서비스 |
| **텍스트 SNS (스레드/X)** | 텍스트 바이럴, 초기 성장 유리 | 정보/인사이트 기반 서비스 |
| **커뮤니티 시딩** | 직접 타겟, 높은 신뢰도 | 특정 연령/관심사 타겟 |
| **검색/SEO** | 장기 자산, 의도 있는 유입 | 정보성 콘텐츠, B2B |
| **유튜브** | 신뢰도 최강, 깊은 설명 | 복잡한 가치 전달 필요 시 |
| **카카오 채널** | 도달률 88%, 열람률 80%+ | 한국 타겟, 리텐션 채널 |
| **유료 광고** | 즉시 트래픽, 비용 발생 | PMF 검증 후 스케일링 |

### 랜딩 설계 원칙

SNS에서 클릭한 사람이 **3초 안에 "해볼까?" 결정**하게 만들어야 한다.

| 원칙 | 설명 |
|------|------|
| **즉시 가치 체험** | 로그인 벽 없이 핵심 기능 체험 가능 |
| **유입 경로별 맞춤 랜딩** | SNS, 검색, 추천 각각 최적화된 첫 화면 |
| **진입 장벽 최소화** | 최소 입력으로 결과 제공 |
| **동적 OG 이미지** | 공유 링크에 개인화된 결과/콘텐츠 자동 표시 |

### 핵심 지표

| 지표 | 정의 |
|------|------|
| **MAU** (Monthly Active Users) | 한 달간 서비스를 이용한 순수 이용자 수 |
| **DAU** (Daily Active Users) | 일간 순수 활동 이용자 수 |
| **NU** (New Users) | 신규 방문자 수 |
| **채널별 유입수** | UTM 파라미터 기반 채널별 클릭 수 |
| **CPC / CPM** | 유료 광고 시 클릭/노출당 비용 |

## 2. Activation — 활성화

> 목표: 처음 들어온 사람이 '가치'를 체감하고 핵심 행동(가입/구독)을 완료한다.
> 핵심 질문: "유저가 '이거 괜찮은데?'라고 느끼는 Aha Moment가 어디인가?"

### Aha Moment 설계

**Aha Moment**: 유저가 서비스의 핵심 가치를 처음 체감하는 순간.
이 순간까지의 시간(**Time to Value**)을 최소화하는 게 Activation의 전부다.

| 서비스 유형 | Aha Moment 예시 |
|------------|----------------|
| 진단/테스트 | AI가 생성한 결과를 보는 순간 |
| SaaS | 첫 번째 작업이 성공하는 순간 |
| 커머스 | 원하는 상품을 발견하는 순간 |
| 콘텐츠 | "이거 나한테 맞는 정보다" 느끼는 순간 |

### 이탈 포인트별 대응 패턴

| 이탈 구간 | 원인 | 대응 |
|----------|------|------|
| 랜딩 → 첫 액션 | 뭘 해야 하는지 모름 | 명확한 CTA ("딱 3초면") |
| 첫 액션 → 입력 | 입력이 귀찮음 | 필드 최소화, 간편 로그인 |
| 입력 → 결과 대기 | 로딩이 길면 이탈 | 기대감 조성 (진행 상태 표시) |
| 결과 → 가입 | 가입 동기 부족 | 결과 저장/공유를 가입 인센티브로 |

### Activation 핵심 전략

1. **Time to Value 최소화** — 클릭 → 핵심 가치 체험까지 **60초 이내** 목표
2. **로그인 벽 제거** — Aha Moment까지는 비회원으로 가능하게
3. **결과의 WOW 퀄리티** — 캡처해서 공유하고 싶은 수준의 결과물
4. **가입 = 보상** — 가입 시 즉시 보상 (크레딧, 프리미엄 체험, 결과 영구 저장)

### 비회원 → 회원 전환 트리거 패턴

| 트리거 | 메커니즘 |
|--------|---------|
| 무료 체험 횟수 제한 | N회 무료 → 가입 시 추가 제공 |
| 결과 휘발성 | 비회원 결과는 저장 불가 → "저장하려면 가입" |
| 프리미엄 기능 해금 | 기본 무료, 상세 분석은 가입 후 |
| 인센티브 보상 | 가입 시 크레딧/쿠폰 즉시 지급 |

### 핵심 지표

| 지표 | 정의 |
|------|------|
| **Bounce Rate** | 첫 페이지에서 서비스를 종료한 비율 (부정적 UX 지표) |
| **PV** (Page View) | 둘러본 페이지 수 (서비스 활성화 지표) |
| **DT** (Duration Time) | 평균 체류 시간 (상호작용 강도 지표) |
| **가입전환율** | (회원가입수 / 방문수) × 100 |
| **Time to Aha** | 첫 방문 → Aha Moment까지 소요 시간 |
| **Engagement Rate** | 특정 핵심 이벤트의 참여율 |

## 3. Retention — 리텐션

> 목표: 한 번 온 사람이 다시 돌아오는 이유를 만든다.
> 핵심: **리텐션 없이 Acquisition에 투자하면 밑 빠진 독에 물 붓기.**

### 리텐션 엔진 유형

| 엔진 | 주기 | 메커니즘 | 예시 |
|------|------|---------|------|
| **주기적 콘텐츠** | 일간/주간 | 정기 콘텐츠 발행 → 알림 → 재방문 | 주간 보고서, 일일 인사이트 |
| **AI/챗봇** | 수시 | 고민/질문이 있을 때 진입 | AI 상담, 챗봇 |
| **성장/수집** | 비정기 | 점진적 진행 → "더 모으고 싶다" | 태그 수집, 레벨업, 배지 |
| **복귀 인센티브** | 이탈 시 | 이탈 유저 복귀 시 보상 | 복귀 쿠폰, 환영 메시지 |
| **소셜/커뮤니티** | 상시 | 다른 유저와의 연결 | 댓글, 랭킹, 그룹 |

### 리텐션 루프 설계 템플릿

```
[주간 루프]
  정기 알림 수신 (알림톡/푸시/이메일)
    → 주간 콘텐츠 확인 → 인터랙션 → 다음 주 기대

[일상 루프]
  필요할 때 진입 (검색, AI, 도구)
    → 무료 사용 → 가치 체감 → 프리미엄 전환

[성장 루프]
  활동 → 보상 축적 → 해금/레벨업
    → "더 모으고 싶다" → 재방문
```

### 리텐션 채널 효과

| 채널 | 열람률 | 클릭률 | 비용 |
|------|--------|--------|------|
| 카카오 알림톡 | **80%+** | 15~30% | 7~10원/건 |
| 푸시 알림 | 40~60% | 5~15% | 무료 |
| 이메일 뉴스레터 | 20~30% | 2~5% | 무료~저가 |
| SMS | 90%+ | 3~8% | 20원+/건 |

### 핵심 지표

| 지표 | 정의 | 공식 |
|------|------|------|
| **Stickiness** | 일상적으로 쓰는 정도 | DAU / MAU |
| **CRR** (Customer Retention Rate) | 고객 유지율 | (기간 종료 시 기존 유저 - 신규 유저) / 기간 시작 시 유저 × 100 |
| **N일차 리텐션** | Day 0 유입자 중 N일차 재방문 비율 | Day N 재방문 수 / Day 0 유입 수 × 100 |

## 4. Revenue — 매출

> 목표: 유저가 돈을 쓸 만큼의 가치를 느끼는 구조를 만든다.
> 핵심: 무료에서 유료로의 자연스러운 업셀링 경로.

### 무료 → 유료 전환 모델

| 모델 | 메커니즘 | 전환율 벤치마크 |
|------|---------|---------------|
| **프리미엄** | 무료 사용 → 자발적 유료 전환 | 3~5% (Good), 6~8% (Great) |
| **Opt-out Trial** | 7일 무료 → 자동 유료 전환 | 30%+ (Good), 49.9% (Great — 카드 정보를 미리 입력한 유저 기준) |
| **마이크로트랜잭션** | 소액 결제 (1,000~5,000원) | 충동구매 유도 (전체 결제의 72%) |
| **크레딧/토큰** | 무료 크레딧 지급 → 소진 → 충전 유도 | 첫 체험 후 자연스러운 과금 |

### 업셀링 퍼널 설계

```
[경로 1: 무료 → 유료 콘텐츠]
  무료 체험 결과 확인
    → 결과 하단 "더 깊은 분석 보기" CTA (감정 최고조 직후)
    → 유료 결제 → 높은 퀄리티 체감 → "또 쓰고 싶다"

[경로 2: 무료 크레딧 → 유료 충전]
  활동 보상으로 무료 크레딧 획득
    → 크레딧으로 유료 기능 첫 체험 (무료 느낌)
    → 퀄리티 체감 → 크레딧 소진 → 충전 유도

[경로 3: 제한 → 해금]
  무료 N회 사용 → 제한 도달
    → "더 쓰고 싶은데" → 결제/구독 전환
```

### 가격 전략

| 전략 | 방법 | 효과 |
|------|------|------|
| **앵커링** | 고가 먼저 보여주고 저가 제시 | 저가 구매율 상승 |
| **마이크로트랜잭션** | 1,000~5,000원 소액 결제 | 충동구매 유도 |
| **번들링** | 개별 3개 = 15만 → 패키지 9만 | 객단가 상승 |
| **연속 할인** | 첫 결제 쿠폰 + 대량 구매 할인 | 첫 전환 + 객단가 동시 상승 |

### PMF 검증 공식

```
3 × CAC < LTV → PMF 달성

CAC = 총 마케팅 비용 / 신규 유저 수
LTV = ARPU × 평균 이용 기간 (월)

오가닉 중심이라 CAC ≈ 0이면 → LTV를 키우는 것이 핵심
LTV 향상 = 리텐션 기간 연장 × 결제 빈도 증가 × 객단가 유지
```

### 핵심 지표

| 지표 | 정의 | 공식 |
|------|------|------|
| **AOV** (Average Order Value) | 평균 주문 금액 | 총 매출 / 총 주문 수 |
| **ARPU** (Average Revenue Per User) | 유저 1인당 평균 매출 | 총 매출 / MAU |
| **ARPPU** (Average Revenue Per Paying User) | 결제 유저 1인당 평균 매출 | 총 매출 / 결제 유저 수 |
| **LTV** (Life-Time Value) | 유저 1명의 생애 잠재 수익 | ARPPU × 평균 이용 기간 |
| **LTR** (Life-Time Revenue) | 유저 1명의 생애 잠재 매출 | ARPU × 평균 이용 기간 |
| **CAC** (Customer Acquisition Cost) | 유저 1명 유치 비용 | 총 마케팅 비용 / NU |
| **GMV** (Gross Merchandise Volume) | 총 거래액 | 판매 상품 가격 × 판매 수량 |
| **구매전환율** | 방문 대비 구매 비율 | (구매건수 / 방문수) × 100 |
| **판매전환율** | 조회 대비 판매 비율 | (판매건수 / 조회수) × 100 |
| **장바구니 전환율** | 장바구니 담기 대비 구매 비율 | (구매건수 / 장바구니 담기 수) × 100 |

## 5. Referral — 추천

> 목표: 유저가 자발적으로 "너도 해봐"라고 말하게 만든다.
> 핵심: 공유하고 싶은 결과물 + 공유하면 보상.

### 공유 동기 유형

| 동기 | 메커니즘 | 예시 |
|------|---------|------|
| **정체성 표현** | "나는 이런 사람이야" 선언 | 유형 카드, MBTI 결과, 컬러 카드 |
| **유머/재미** | "이거 봐ㅋㅋ" | 밸런스 결과, 점수, 밈 |
| **유용함** | "이거 너한테 필요할 듯" | 가이드, 도구, 꿀팁 |
| **인센티브** | "공유하면 둘 다 보상" | 양방향 크레딧, 쿠폰 |
| **비교/경쟁** | "우리 94%래!" | 궁합, 점수, 랭킹 |

### Referral 루프 설계

```
[자발적 공유 루프]
  결과 확인 → 결과물이 예쁘고 "나를 잘 설명함"
    → 캡처/링크 → SNS 공유 → 친구 유입 → 친구도 결과 공유 → 바이럴

[인센티브 공유 루프]
  공유 버튼 클릭 → 친구에게 링크 발송
    → 친구 가입 → 친구에게 보상 지급
    → 나에게도 보상 지급 → 양쪽 모두 유료 기능 체험
    → "나도 해봤는데 진짜 좋아" → 추가 공유

[강제 2인 루프 (최강)]
  제품 자체가 공유 없이 작동 불가
    → 궁합/매칭/퀴즈 → 상대방 참여 필수 → 자동 바이럴
```

### K-Factor (바이럴 계수)

```
K = (사용자당 평균 초대 횟수) × (추천 전환율)

K < 0.5  → 자연 성장 거의 없음
K = 0.5  → 느린 성장
K > 1.0  → 자가 증식 (마케팅 비용 없이 유저가 유저를 데려옴)
```

**K-Factor 올리는 방법**:
- **초대 횟수 늘리기** → 공유하고 싶은 결과물 퀄리티 + 공유 접점 확대
- **전환율 올리기** → 공유 링크의 랜딩 퀄리티 + 진입장벽 제거

### 핵심 지표

| 지표 | 정의 | 공식 |
|------|------|------|
| **K-Factor** | 바이럴 계수 | (사용자당 평균 초대 횟수) × (추천 전환율) |
| **NPS** (Net Promoter Score) | 순추천지수 | 추천고객 비율 - 비추천고객 비율 |
| **공유 클릭률** | 활성 유저 중 공유 버튼 클릭 비율 | 공유 클릭 수 / 활성 유저 수 × 100 |
| **초대 링크 전환율** | 공유 링크 클릭 → 가입 완료 비율 | 링크 경유 가입 수 / 링크 클릭 수 × 100 |

## 퍼널 간 연결 — 플라이휠

```
[Acquisition → Activation]
  유입 → 비회원 무료 체험 → Aha Moment

[Activation → Retention]
  가입 완료 → 보상 지급 → 주기적 알림 시작 → 매주 재방문 이유 생성

[Retention → Revenue]
  무료 크레딧으로 유료 첫 체험 → 퀄리티 체감 → 추가 결제

[Revenue → Referral]
  유료 결과의 높은 퀄리티 → "이거 진짜 좋다" → 자발적 공유 + 인센티브 공유

[Referral → Acquisition]
  공유된 링크/카드 → 새 유저 클릭 → 퍼널 자동 순환
```

## 퍼널 우선순위 판단 기준

지금 어디에 집중해야 하는가:

| 상황 | 우선순위 | 이유 |
|------|---------|------|
| 유저가 안 들어옴 | Acquisition | 유입 자체가 없으면 퍼널이 시작 안 됨 |
| 들어오는데 바로 나감 | Activation | Aha Moment까지 도달 못 함 |
| 가입하는데 재방문 없음 | **Retention** | 리텐션 없이 성장 = 밑 빠진 독 |
| 재방문하는데 결제 안 함 | Revenue | 무료→유료 퍼널 막힘 |
| 결제하는데 성장이 느림 | Referral | 오가닉 바이럴 루프 부재 |

> **일반적으로 Retention → Activation → Referral → Revenue → Acquisition 순서가 임팩트가 크다.**
> 새는 물을 먼저 막고(리텐션), 들어온 물을 잡고(활성화), 퍼지게 하고(추천), 수익화하고(매출), 그 다음 더 넣는다(유입).

## 지표 전체 요약 (빠른 참조)

### Acquisition
| MAU | DAU | NU | CPC | CPM |

### Activation
| Bounce Rate | PV | DT | 가입전환율 | Time to Aha | Engagement Rate |

### Retention
| Stickiness (DAU/MAU) | CRR | N일차 리텐션 |

### Revenue
| AOV | ARPU | ARPPU | LTV | LTR | CAC | GMV | 구매전환율 | 판매전환율 | 장바구니 전환율 |

### Referral
| K-Factor | NPS | 공유 클릭률 | 초대 링크 전환율 |

### PMF 기준
```
3 × CAC < LTV → Product-Market Fit 달성
```

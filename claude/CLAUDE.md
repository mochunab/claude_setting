# Global Rules

## 응답 스타일
- 간결하게 답변. 코멘트는 로직이 자명하지 않을 때만
- 한국어 기본. 코드/커밋 메시지는 영어 OK

## 아키텍처 원칙
- 객체지향적 모듈화 우선 — 결합도↓ 응집도↑, 책임이 명확한 단위로 분리
- 작은 수정이라도 확장 가능한 구조인지 먼저 검토
- 변경 시 해당 파일만 보지 말고 전체 흐름(입력→처리→저장→출력→발행) 점검
- 새 컴포넌트 전 기존 컴포넌트/유틸 확인 — 중복 생성 방지

## 퍼포먼스
- 독립적인 비동기 작업은 순차 `await` 대신 `Promise.all`로 병렬 실행
- Provider/Context에서 이미 제공하는 데이터 재활용 — 중복 API/DB 호출 금지
- 미들웨어/가드에 네트워크 호출 금지 — 순수 로직만
- 캐시 우선 렌더링 — 캐시 데이터로 즉시 표시 후 백그라운드 갱신 (stale-while-revalidate)
- 자주 접근하는 데이터는 인메모리 캐시 + TTL, 데이터 변경 시 캐시 무효화

## TypeScript
- `any` 타입 금지
- 외부 API/DB 응답에 타입 체크 필수

## 보안
- 시크릿/API 키 하드코딩 금지 — 반드시 환경변수
- `error.message` 사용자 직접 노출 금지 — 일반 메시지만 표시, 상세는 내부 로그
- 입력값은 서버에서 재검증 — 클라이언트 검증만으로 신뢰하지 않기

## 환경 분리
- 개발/프로덕션 환경 플래그 체계화 — 디버그 코드가 프로덕션에 노출되지 않도록

## 문서
- CLAUDE.md는 200줄 이내 유지 — 코드에서 유추 가능한 것은 적지 않음
- 커밋: `<type>: <description>` — feat, fix, docs, style, refactor, test, chore
